# Tachyon Cowork 手動動作確認ガイド

## 1. 前提条件

### 必要な環境・ツール

| ツール                   | バージョン     | 用途                                    |
| ------------------------ | -------------- | --------------------------------------- |
| **Rust** (rustc + cargo) | stable 最新    | Tauri バックエンド (Rust) のビルド      |
| **Node.js**              | v20 以上推奨   | フロントエンド (Vite + React) のビルド  |
| **npm**                  | Node.js 付属版 | パッケージ管理                          |
| **Tauri CLI**            | v2 系          | `cargo tauri dev` / `cargo tauri build` |
| **システムライブラリ**   | OS 依存        | 下記参照                                |

#### システムライブラリ (Linux)

```bash
# Ubuntu / Debian
sudo apt install -y \
  libwebkit2gtk-4.1-dev libgtk-3-dev libayatana-appindicator3-dev \
  librsvg2-dev patchelf libssl-dev
```

#### システムライブラリ (macOS)

Xcode Command Line Tools がインストールされていれば追加ライブラリは不要。

```bash
xcode-select --install
```

#### Tauri CLI インストール

```bash
cargo install tauri-cli --version "^2"
```

## 2. セットアップ手順

### 2.1 リポジトリの取得

```bash
git clone https://github.com/quantum-box/tachyon-cowork.git
cd tachyon-cowork
```

### 2.2 依存関係のインストール

```bash
npm install
```

### 2.3 環境変数の設定

`.env.example` をコピーして `.env` を作成する。

```bash
cp .env.example .env
```

`.env` の内容 (デフォルト値):

```env
VITE_COGNITO_DOMAIN=https://auth-pool.n1.tachy.one
VITE_COGNITO_CLIENT_ID=78a4raqiqns509aadtv7ftjmee
VITE_COGNITO_REDIRECT_URI=http://localhost:1420/callback
VITE_COGNITO_SCOPES=openid email profile
VITE_API_BASE_URL=https://api.n1.tachy.one
VITE_DEFAULT_TENANT_ID=tn_01hjryxysgey07h5jz5wagqj0m
```

- `VITE_COGNITO_REDIRECT_URI` は開発時は `http://localhost:1420/callback` のままでよい
- 本番 / ステージング API に向ける場合は `VITE_API_BASE_URL` と `VITE_DEFAULT_TENANT_ID` を変更する

## 3. 起動方法

### 3.1 開発モードで起動

```bash
cargo tauri dev
```

これにより以下が順次実行される:

1. `npm run dev` (Vite 開発サーバーが `127.0.0.1:1420` で起動)
2. Rust バックエンドのビルド
3. Tauri ウィンドウが開く

初回はRustクレートのコンパイルに数分かかる。

### 3.2 トラブルシューティング

#### ポート 1420 が使用中

```
Error: Port 1420 is already in use
```

対処: 既存プロセスを終了するか、別ポートを指定する。

```bash
# 1420 を使っているプロセスを確認
lsof -i :1420
# 別ポートで起動する場合は package.json の dev スクリプトと tauri.conf.json の devUrl を揃えて変更
```

#### Rust ビルドエラー: tachyon-sdk の取得失敗

```
error: failed to fetch `https://github.com/quantum-box/tachyon-sdk.git`
```

対処: GitHub へのアクセス権限を確認する。SSH 設定または Personal Access Token が必要。

```bash
# SSH の場合
ssh -T git@github.com
# HTTPS の場合
git config --global credential.helper store
```

#### WebKit / GTK 関連エラー (Linux)

対処: 前提条件のシステムライブラリをインストールする。

#### `microsandbox` 関連の警告

Sandbox 機能は microsandbox を使用する。ローカル開発では microsandbox デーモンが起動していなくても、sandbox 以外の機能は正常に動作する。

## 5. 既知の問題・制約

### 認証関連

- **deep-link スキーム**: デスクトップアプリでは `tachyon-cowork://` カスタムスキームが登録されるが、開発モードでは localhost リダイレクトを使用する
- **トークンリフレッシュ**: バックグラウンドで自動リフレッシュされるが、長時間放置するとリフレッシュトークンも期限切れになる場合がある

### サンドボックス

- **microsandbox 依存**: コード実行機能は microsandbox デーモンが必要。未起動の場合、コード実行ボタンは機能しない
- **サポート言語**: microsandbox がサポートする言語に依存する

### MCP

- **プロセスベース MCP**: MCP サーバーは子プロセスとして起動される (`rmcp` クレート使用)。対応サーバーのバイナリがローカルに必要
- **Built-in Apps**: `file_manager`, `office_worker`, `web_search` が組み込み MCP App として用意されているが、一部は外部 API キーが必要な場合がある

### ファイル操作

- **PDF テキスト抽出**: 画像ベースの PDF (スキャン PDF) からのテキスト抽出は対応していない (`lopdf` ベース)
- **大容量ファイル**: 非常に大きなファイルの添付は処理時間がかかる場合がある

### ビルド・開発

- **tachyon-sdk**: プライベートリポジトリのため、GitHub アクセス権限が必要
- **初回ビルド時間**: Rust の依存クレートのコンパイルに 3-5 分程度かかる
- **ホットリロード**: フロントエンド (Vite) はホットリロード対応。Rust 側の変更は再コンパイルが必要

### UI

- **ウィンドウサイズ**: 最小サイズは 800x600。これ以下ではレイアウトが崩れる可能性がある
- **ブラウザ互換性**: Tauri の WebView を使用するため、OS の WebView エンジンに依存する (Linux: WebKitGTK, macOS: WKWebView, Windows: WebView2)
