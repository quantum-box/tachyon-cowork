# Cowork 業務ユースケース改善メモ

## PLT-419 対象

KR3「Cowork週5使用」を想定し、以下 4 ユースケースを実用レベルへ引き上げるためのブロッカーと改善を整理した。

## 1. 日報作成

### 変更前のブロッカー

- 汎用テンプレートしかなく、日報として必要な出力項目が揃わない
- 作業ログや添付メモをそのまま日報へ落とし込む導線が弱い

### 実装した改善

- チャットのクイックテンプレートに `日報作成` を追加
- `本日の実績 / 完了した作業 / 課題 / 明日の予定 / 共有事項` を固定化
- 添付したテキスト・Office 文書・PDF の内容をローカル抽出し、日報生成の入力文脈に自動で取り込む

## 2. 議事録

### 変更前のブロッカー

- 既存テンプレートは議事録の粒度が粗く、担当者・期限・保留事項が落ちやすい
- 会議メモ添付時に中身を確実に読ませる仕組みが弱い

### 実装した改善

- `議事録整形` テンプレートを追加
- `会議名 / 日時 / 参加者 / 議題 / 決定事項 / 保留事項 / アクションアイテム / 次回までの宿題` を固定化
- PDF / DOCX / XLSX / PPTX / CSV をローカルで抽出してからプロンプトへ埋め込む

## 3. コード相談

### 変更前のブロッカー

- Tauri のファイル添付で MIME type とローカルパスが落ちる
- コードを base64 添付するだけで、モデルが十分に内容を読めないケースがある
- 添付したコードをどう読んでほしいかの指示が弱い

### 実装した改善

- ファイルピッカー経由の添付で MIME type と絶対パスを保持
- `.ts/.tsx/.js/.py/.rs/.md/.json` などのコード・テキスト系ファイルを送信前にローカル抽出
- コード相談時は `結論 / 根拠 / 修正案 / 追加確認点` を促す専用テンプレートを追加
- 抽出したファイル内容に `Path` を付け、必要ならローカルツールで追加確認できる形にした

## 4. ファイル整理

### 変更前のブロッカー

- ファイル検索は件数上限が低く、大量ファイルで探索しづらい
- 検索が常時深さ 10 固定で、再帰・隠しファイル・件数制御ができない
- 重複検出は候補ファイルを丸ごとメモリに読み込んでハッシュしており、大きいファイル群で重い
- 整理プランは直下ファイルしか対象にできず、実務ディレクトリで使いにくい
- 別ボリューム間の移動で `rename` が失敗しうる

### 実装した改善

- ファイル検索に `再帰検索 / 隠しファイル / 件数上限 200-2000` を追加
- 検索対象をファイル名だけでなくパス文字列にも拡張
- 重複検出を `サイズ -> 先頭/末尾サンプルハッシュ -> フルハッシュ` の段階式に変更
- フルハッシュはストリーミング化し、大きいファイルでも全読込を避ける
- ファイル整理に `サブフォルダも整理対象に含める` オプションを追加
- 移動処理は `rename` 失敗時に `copy + remove` へフォールバック

## 手動確認の観点

- 空チャットで `日報作成` `議事録整形` `コード相談` テンプレートが見える
- `.md` や `.ts` を添付してコード相談すると、添付内容が回答根拠に反映される
- `.docx` `.xlsx` `.pptx` `.pdf` を添付した議事録・日報生成で内容が回答に反映される
- ファイル検索で大きめディレクトリに対し、再帰有無と件数上限を切り替えられる
- 重複検出で大容量ファイル群をスキャンしてもブラウザ側が固まらない
- ファイル整理でサブフォルダを含むプランが作成でき、別ボリュームでも移動に失敗しにくい
